# 质量评分系统

**目标**: 量化代码质量，追踪改进，确保智能体生成的代码保持一致标准

---

## 一、评分维度

### 1. 代码质量 (Code Quality) — 权重 30%

| 指标 | 优秀 (100) | 良好 (80) | 需改进 (60) | 差 (<60) |
|------|-----------|----------|-----------|---------|
| 类型安全 | 100% strict, 无 any | <5% any | <10% any | >10% any |
| 测试覆盖 | 100% | 90-99% | 80-89% | <80% |
| 文件大小 | 全部 ≤500 行 | <5% 超限 | <10% 超限 | >10% 超限 |
| 函数长度 | 全部 ≤50 行 | <5% 超限 | <10% 超限 | >10% 超限 |
| Lint 规则 | 0 violations | <5 warnings | <10 warnings | >10 warnings |

### 2. 架构一致性 (Architecture) — 权重 25%

| 指标 | 优秀 (100) | 良好 (80) | 需改进 (60) | 差 (<60) |
|------|-----------|----------|-----------|---------|
| 依赖边界 | 0 违规 | <3 违规 | <5 违规 | >5 违规 |
| 分层清晰 | 100% 符合 | <3% 违规 | <5% 违规 | >5% 违规 |
| 模块耦合 | 低耦合 | 中等 | 较高 | 高耦合 |
| 命名一致 | 100% 一致 | <5% 不一致 | <10% 不一致 | >10% 不一致 |

### 3. 文档完整性 (Documentation) — 权重 20%

| 指标 | 优秀 (100) | 良好 (80) | 需改进 (60) | 差 (<60) |
|------|-----------|----------|-----------|---------|
| AGENTS.md | ≤100 行, 最新 | ≤120 行 | ≤150 行 | >150 行 |
| 设计文档 | 全部 active, 最新 | <10% 过时 | <20% 过时 | >20% 过时 |
| API 文档 | 100% 覆盖 | 90% 覆盖 | 80% 覆盖 | <80% 覆盖 |
| 交叉链接 | 全部有效 | <5% 死链 | <10% 死链 | >10% 死链 |

### 4. 可维护性 (Maintainability) — 权重 15%

| 指标 | 优秀 (100) | 良好 (80) | 需改进 (60) | 差 (<60) |
|------|-----------|----------|-----------|---------|
| 技术债务 | 0 高优先级 | <3 高优先级 | <5 高优先级 | >5 高优先级 |
| 代码重复 | <3% | <5% | <10% | >10% |
| 构建时间 | <30s | <60s | <120s | >120s |
| CI 通过率 | 100% | >95% | >90% | <90% |

### 5. 可靠性 (Reliability) — 权重 10%

| 指标 | 优秀 (100) | 良好 (80) | 需改进 (60) | 差 (<60) |
|------|-----------|----------|-----------|---------|
| 错误处理 | 100% 覆盖 | 90% 覆盖 | 80% 覆盖 | <80% 覆盖 |
| 重试机制 | 全部有重试 | 关键路径有 | 部分有 | 无重试 |
| 日志完整 | 结构化日志 | 大部分有 | 部分有 | 无日志 |

---

## 二、计算公式

```
总分 = (代码质量 × 0.30) + (架构一致性 × 0.25) + (文档完整性 × 0.20) 
     + (可维护性 × 0.15) + (可靠性 × 0.10)
```

### 等级划分

| 等级 | 分数 | 操作 |
|------|------|------|
| A (优秀) | 90-100 | 自动合并 PR |
| B (良好) | 80-89 | 正常合并 PR |
| C (需改进) | 70-79 | 需要审查后合并 |
| D (差) | 60-69 | 阻止合并，必须修复 |
| F (失败) | <60 | 阻止合并，紧急修复 |

## 三、自动化检查

### CI 门禁

| 检查 | 命令 | 失败条件 |
|------|------|----------|
| 类型检查 | `pyright src/agents_mem` | 任何错误 |
| 测试 | `pytest` | 任何失败 |
| 代码风格 | `ruff check src/agents_mem` | 任何违规 |
| 文档检查 | 手动验证链接 | 任何 error |

### 定期扫描

- **每日**: doc-gardening 智能体扫描过时文档
- **每周**: 技术债务评估 + 质量评分更新
- **每月**: 架构一致性全面审查
