当然有，软件测试的类型非常多，我按**常见分类**给你整理一份清晰的清单，顺便把你提到的三种也放进去对照：

# 一、按测试阶段
1. **单元测试**
   对代码最小单元（函数、方法）进行测试，由开发为主。
2. **集成测试**
   测试多个模块/接口组合后是否能正常协作。
3. **系统测试**
   对整个完整系统做整体功能、流程验证。
4. **验收测试**
   交付前最终验证，包括：
   - UAT（用户验收测试）
   - α测试（内部内测）
   - β测试（外部公测）

# 二、按测试策略/方式
5. **冒烟测试**
   快速验证核心流程是否通，决定是否进入正式测试。
6. **回归测试**
   修改代码后，重新测试原有功能，防止引入新bug。
7. **探索性测试**
   无用例，凭经验自由操作找问题。
8. **随机测试**
   随机输入、随机操作，模拟异常场景。

# 三、按测试技术视角
9. **黑盒测试**
   不看代码，只看输入输出。
10. **白盒测试**
    基于代码逻辑、分支覆盖测试。
11. **灰盒测试**
    结合黑盒+白盒，常见于接口测试。

# 四、按测试目标/特性
12. **功能测试**
    验证功能是否符合需求。
13. **性能测试**
    - 负载测试
    - 压力测试
    - 并发测试
    - 稳定性测试
14. **安全测试**
    漏洞扫描、渗透测试、防注入、权限测试等。
15. **兼容性测试**
    不同浏览器、系统、版本、分辨率下是否正常。
16. **易用性测试**
    体验、操作流程是否友好。
17. **接口测试**
    专门测 API、HTTP/HTTPS 接口。
18. **UI测试**
    页面布局、样式、交互是否正常。
19. **可靠性/稳定性测试**
    长时间运行是否崩溃、内存泄漏等。

# 五、专项测试
- 移动端专项：弱网测试、电量/流量测试、安装卸载测试
- 大数据测试、并发测试、灾备测试、国际化/本地化测试
- 自动化测试、压力测试、混沌测试等
