# 核心理念 — 智能体优先

**原则**: 人类掌舵，智能体执行

---

## 1. 代码仓库是记录系统

- 智能体只能访问代码仓库内的版本化工件
- Google Docs、Slack、隐性知识 = 对智能体不存在
- 所有知识必须以 Markdown 形式编码到仓库中

## 2. 给地图，不给说明书

- `AGENTS.md` 是内容目录 (~100 行)，不是百科全书
- 知识库位于结构化的 `docs/` 目录
- 渐进式披露：从小切入点开始，指导下一步查看

## 3. 为智能体可读性优化

- 代码首先对 Codex 可读，其次对人类可读
- 选择"枯燥"但可组合、API 稳定、易建模的技术
- 边界处验证数据形状 (Parse, don't validate)

## 4. 强制不变量，不微观管理实现

- 通过自定义 linter 和结构测试强制执行
- 依赖方向严格验证，仅允许有限的边
- 边界内允许自主权

## 5. 纠错成本低，等待成本高

- PR 生命周期短
- 测试偶发失败通过重跑解决，不无限阻塞
- 智能体吞吐量远超人类注意力

## 6. 垃圾回收是持续的

- 技术债务 = 高息贷款
- 定期扫描偏差、更新质量等级、发起重构 PR
- 人类品味一旦捕捉，就持续应用于每一行

## 7. 文档必须可验证

- 专职 linter 和 CI 验证知识库更新、交叉链接、结构正确
- doc-gardening 智能体扫描过时文档
- 不再反映代码行为的文档必须修复或删除
